# tx
tx - CLI for semantic models
  opt --help, -?, -h, /?, /h  Show help and usage information
  opt --version  Show version information
  opt --model, -m <model>  Path to semantic model (TMDL folder, .bim file, or TE folder)
  opt --output-format <output-format>  Stdout format: text (default), json, csv, tmsl (alias: bim), tmdl. Not all formats are supported by every command.
  opt --error-format <error-format>  Stderr format for errors/warnings/hints: text (default) or json. Other values fall back to text.
  opt --server, -s <server>  Workspace name or endpoint (e.g., MyWorkspace, powerbi://..., asazure://..., localhost)
  opt --database, -d <database>  Semantic model name on the workspace
  opt --local  Connect to a locally running Power BI Desktop instance (Windows only)
  opt --auth <auth>  Auth method: auto, interactive, spn, managed-identity (default: auto)
  opt --recent, --recents <recent>  Use a recently used model. No value = interactive picker, N = Nth most recent (1 = last used).
  opt --debug  Enable debug logging to stderr (connection strings, auth flow, timing)
  opt --non-interactive  Disable all interactive prompts. Fail with an actionable error if required input is missing.
  opt --yes, -y  Skip confirmation prompts for destructive operations
  opt --quiet  Suppress non-essential output (spinners, progress, hints). Errors and data still print.

# tx add
Add an object to the model
  arg <path>  Object path of the new object. Slash-separated: 'Sales/Revenue', 'Products', 'Admin'. DAX forms also accepted: "'Sales'[Revenue]". Pair with -t. Relationships use 'Sales[Key]->Product[Key]' (many side -> one side).
  arg [model]  Path to model (if not using --model)
  opt --type, -t <type>  Object type. Known values: Table, CalcTable, CalcGroup, Measure, CalcColumn, DataColumn, Hierarchy, Level, Calendar, CalcItem, KPI, Partition, MPartition, EntityPartition, PolicyRangePartition, Expression, Function, Perspective, Culture, ProviderDataSource, StructuredDataSource, Role, TablePermission, Member, Relationship. Data sources always require -t (no path keyword infers them).
  opt -i <i>  Expression or value for the new object. Use '-' to read from stdin. When paired with a preceding -q, applies as that property's value -- so you can set extra properties on the new object in one command, e.g. -i "SUM(...)" -q description -i "my measure" -q formatString -i "$#,0".
  opt -q <q>  Property name to set on the newly-created object. Pair each -q with a following -i value. Repeatable. Names accept dotted paths, bracket indexers, and DisplayName matching.
  opt --file <file>  Read expression from file
  opt --if-not-exists  Succeed silently if the object already exists (exit 0)
  opt --force  Save even if this mutation introduces validation errors
  opt --save-to <save-to>  Save to a different path (implies --save)
  opt --serialization <serialization>  Model serialization: tmdl, bim (tmsl and auto also accepted)
  opt --save  Persist this command's mutation to the source location. Mutually exclusive with --revert and --stage.
  opt --stage  Stage this command's mutation
  opt --revert  Revert a staged mutation
  opt --no-sync  Skip workspace sync when workspace mode is active.
  opt --mode <mode>  Partition storage mode: Import, DirectQuery, Dual, DirectLake, Push, Default.
  opt --source <source>  Provider name for a ProviderDataSource (e.g. System.Data.SqlClient).
  opt --endpoint <endpoint>  Server/endpoint address for a data source connection.
  opt --connection-string <connection-string>  Full connection string for a ProviderDataSource.
  opt --source-table <source-table>  Source entity/table name for an EntityPartition.
  opt --source-database <source-database>  Source database name for a data source connection (Provider/Structured data sources).
  opt --source-schema <source-schema>  Source schema name for an EntityPartition.
  opt --partition-expression <partition-expression>  M/DAX expression for a partition source.
  opt --columns <columns>  Comma-separated column names to create on a new table (Table type only).
  opt --source-type <source-type>  Connection protocol for a StructuredDataSource (e.g. tds).
  opt --range-start <range-start>  Refresh-policy range start for a PolicyRangePartition (yyyy-MM-dd).
  opt --range-end <range-end>  Refresh-policy range end for a PolicyRangePartition (yyyy-MM-dd).
  opt --range-granularity <range-granularity>  Refresh-policy range granularity for a PolicyRangePartition: Day (default), Month, Quarter, Year.

# tx connect
Set active connection (workspace, local path, or PBI Desktop). No args = show current. --recent = reconnect to a recently used model.
  arg [server]  Workspace name, endpoint, local model path (TMDL/BIM), or omit with --local.
  arg [database]  Semantic model name (omit on a TTY to pick from the workspace's models).
  opt --workspace, -w <workspace>  Enable workspace mode: mirror saves between the primary source and a secondary target. Primary remote -> pass a local folder or .bim file. Primary local -> pass <server> <database>. Pass -w with no value to pick the target workspace and model interactively.
  opt --remote  Pick a workspace and semantic model interactively from your Power BI tenant (requires a TTY; sign in first with 'tx auth login').
  opt --profile, -p <profile>  Activate a saved connection profile (see: tx profile list)
  opt --clear  Clear the active connection from CLI config
  opt --force  Overwrite a non-empty workspace target when initializing workspace mode.
  opt --workspace-format <workspace-format>  On-disk format for a local workspace (tmdl, bim, te-folder). Defaults to path-detected.
  opt --workspace-auth <workspace-auth>  Auth method for a remote workspace (local primary). Defaults to --auth if set, else auto.

  opt --deploy-full  Full deploy: overwrite + connections + partitions + shared expressions + roles + role members
  opt --create-only  Only create new model; fail if it already exists
  opt --deploy-connections  Deploy data source connections
  opt --deploy-partitions  Deploy partition definitions
  opt --skip-refresh-policy  Skip overwriting partitions with Incremental Refresh Policies (use with --deploy-partitions)
  opt --deploy-shared-expressions  Deploy (overwrite) shared expressions
  opt --deploy-roles  Deploy security roles
  opt --deploy-role-members  Deploy role members
  opt --xmla <xmla>  Generate XMLA/TMSL script to file instead of deploying. Use '-' for stdout.
  opt --skip-bpa  Skip BPA gate check (configured via .te-bpa.json)
  opt --fix-bpa  Auto-fix BPA violations before deploying (applies FixExpressions where available)
  opt --bpa-rules <bpa-rules>  Path(s) to BPA rule file(s) for this deploy. Overrides bpa.rules in CLI config.

  opt --ci <ci>  Emit CI logging commands to stderr: vsts (Azure DevOps), github (GitHub Actions)

  opt --skip-validation  Skip DAX semantic validation. Faster for pure download-and-save scenarios.
  opt --supporting-files  Wrap output in a {modelName}.SemanticModel/ folder with .platform and definition.pbism. Only for tmdl/bim on bare targets.
  opt --no-sync  Skip workspace sync when workspace mode is active.

# tx test
Run DAX regression tests against a live model (--update records snapshots, --trx/--ci for pipelines)
  arg [path]  Test file or directory searched recursively for .dax tests (default: current directory)
  opt --update  Record mode: run each query and (re)write its .expected.json snapshot from the actual result.
  opt --filter <filter>  Run only tests whose name matches a * wildcard pattern (case-insensitive).
  opt --param <param>  Query parameter as name=value, referenced as @name in DAX. Applied to every test. Repeatable.
  opt --max-rows <max-rows>  Per-query row cap; a query exceeding it fails as an error (default: 10000).
  opt --ci <ci>  Emit CI logging commands to stderr: vsts or github
  opt --trx <trx>  Write results as a VSTEST .trx file to the specified path

# tx vertipaq
Analyze VertiPaq storage statistics for a semantic model
  arg [table]  Table name to filter the analysis to a single table
  opt --tables  Show the tables view
  opt --columns  Show the columns view (default)
  opt --relationships  Show the relationships view
  opt --partitions  Show the partitions view
  opt --all  Show tables, columns, relationships, and partitions
  opt --detail  Expanded columns: data/dictionary/hierarchy size breakdown, encoding, segments
  opt --fields <fields>  Comma-separated fields to display (e.g. name,card,size,%tbl,%db,bar). Available fields vary by view; requires a single view. Text and csv output only.
  opt --top <top>  Limit each view to the N largest rows. Text and csv output only.
  opt --stats  Show a model-level storage summary (combine with view flags for more)
  opt --annotate  Write statistics into the model as Vertipaq_* annotations (preview unless --save)
  opt --save  Persist --annotate changes to the model
  opt --export <export>  Export statistics to a .vpax file for offline analysis
  opt --import <import>  Analyze a previously exported .vpax file offline (no connection needed)
  opt --obfuscate  Obfuscate names and expressions in the exported .vpax; writes a private .dict dictionary
